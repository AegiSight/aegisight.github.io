Umrah & Haji Guide

Kebijakan Privasi

Terakhir diperbarui: April 30, 2026

Pengembang: Lumina Dev
Situs web: https://lumina-dev-lab.github.io/umrah-and-haji/
Kontak: machealmaxin83@gmail.com

Kebijakan Privasi ini menjelaskan bagaimana Umrah & Haji Guide menangani informasi saat Anda menggunakan aplikasi seluler di iOS dan Android. Aplikasi ini dikembangkan oleh Lumina Dev.

1. Informasi yang Kami Tangani

  • Aplikasi ini dirancang offline-first. Sebagian besar informasi yang Anda buat disimpan di perangkat Anda.
  • Informasi di perangkat dapat mencakup progres ritual, sesi penghitung Tawaf dan Sa’i, du’a tersimpan, item checklist, pin peta tersimpan, kontak darurat, detail keselamatan grup, bahasa, ukuran teks, tema, status paket offline, status pembelian, dan riwayat dukungan kopi.
  • Jika Anda memakai fitur peta, Qibla, atau lokasi saya, aplikasi dapat meminta izin lokasi untuk menghitung arah, menampilkan posisi, menyimpan pin, atau membantu orientasi.
  • Jika Anda memakai transfer QR dukungan kopi, aplikasi dapat meminta akses kamera untuk memindai kode QR.
  • Jika Anda melakukan pembelian dalam aplikasi, toko aplikasi dan layanan pembelian dapat memproses informasi transaksi. Aplikasi dapat menyimpan status pembelian dan ID pendukung anonim untuk pemulihan atau verifikasi pembelian.
  • Informasi persetujuan iklan dan penayangan iklan dapat ditangani melalui Google AdMob dan Google User Messaging Platform jika tersedia.
  • Di Android, Firebase Analytics dapat mengumpulkan informasi interaksi aplikasi dan perangkat seperti pembukaan aplikasi, sesi, model perangkat, sistem operasi, perkiraan wilayah, pembaruan aplikasi, dan peristiwa pembelian dalam aplikasi.
  • Di Android, Firebase Crashlytics dapat mengumpulkan laporan crash, stack trace, status aplikasi terkait, metadata perangkat, ID instalasi Firebase, dan ID instalasi Crashlytics untuk membantu mendiagnosis crash dan masalah stabilitas.

2. Cara Kami Menggunakan Informasi

  • Untuk menyediakan panduan ritual, konten offline, penghitung, du’a, checklist, peta, Qibla, pengaturan darurat, dan fitur aplikasi lainnya.
  • Untuk menyimpan preferensi dan progres Anda secara lokal.
  • Untuk menampilkan peta, tempat tersimpan, paket peta offline, dan orientasi berbasis lokasi saat Anda memberi izin.
  • Untuk mendukung pembelian dalam aplikasi, penghapusan iklan, riwayat dukungan kopi, dan transfer riwayat dukungan antarperangkat jika tersedia.
  • Untuk menampilkan iklan hanya di Home dan Map, serta menghormati pilihan persetujuan iklan.
  • Untuk memahami penggunaan fitur secara umum, kesehatan aplikasi, dan stabilitas, serta mendiagnosis crash atau error melalui Firebase Analytics dan Firebase Crashlytics di Android.

3. Penyimpanan Lokal

Aplikasi menggunakan penyimpanan lokal perangkat, termasuk Isar, SharedPreferences, penyimpanan aman, dan file paket offline. Data lokal tetap berada di perangkat kecuali Anda membagikannya, mentransfernya, memakai fitur backend opsional yang dikonfigurasi, menghapus data aplikasi, atau menghapus aplikasi.

4. Layanan Pihak Ketiga

  • Aplikasi dapat menggunakan Google AdMob untuk iklan, Google User Messaging Platform untuk pilihan persetujuan, Firebase Analytics untuk analitik penggunaan, Firebase Crashlytics untuk pelaporan crash, Mapbox untuk peta dan sumber daya peta offline, sistem pembelian toko aplikasi, dan layanan backend opsional seperti REST API atau Supabase untuk verifikasi dan transfer dukungan kopi jika dikonfigurasi.
  • Layanan tersebut dapat memproses data berdasarkan kebijakan privasi masing-masing.

5. Lokasi, Kamera, dan Berbagi

Akses lokasi bersifat opsional dan dapat diubah di pengaturan perangkat. Kamera digunakan untuk pemindaian QR hanya saat Anda memilihnya. Tindakan berbagi bawaan dapat mengirim titik temu, pin tersimpan, hotel, bus, kemah, pemandu, atau detail serupa ke aplikasi atau orang yang Anda pilih.

6. Iklan

Iklan dapat muncul di Home dan Map, yang merupakan area non-sakral dan bukan sesi ritual aktif. Aplikasi dirancang agar tidak menampilkan iklan selama sesi hitung Tawaf atau Sa’i yang aktif dan tidak mengganggu alur ritual.

7. Analitik dan Pelaporan Crash

Firebase Analytics dan Firebase Crashlytics digunakan di Android untuk memahami penggunaan aplikasi secara umum dan meningkatkan keandalan. Kami tidak sengaja menggunakan alat ini untuk mengumpulkan catatan ritual, detail kontak darurat, tempat tersimpan, catatan medis, atau konten pribadi bebas yang Anda masukkan. Data Firebase diproses oleh Google berdasarkan ketentuan privasi dan praktik retensi Google dan Firebase.

8. Anak-anak

Aplikasi ini tidak dirancang khusus untuk anak-anak. Jika Anda yakin ada informasi yang diberikan secara tidak tepat, hubungi kami.

9. Retensi dan Penghapusan

Data lokal tetap ada hingga Anda menghapusnya di aplikasi, menghapus paket offline, menghapus data aplikasi, atau mencopot aplikasi. Catatan pembelian juga dapat disimpan oleh toko aplikasi atau layanan backend opsional sesuai kebijakannya.

10. Perubahan

Kami dapat memperbarui Kebijakan Privasi ini dari waktu ke waktu. Versi terbaru akan diposting di halaman ini dengan tanggal terbaru.
